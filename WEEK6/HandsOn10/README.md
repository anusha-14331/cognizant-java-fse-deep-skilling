# Hands-On 10 – Unit Testing Angular Applications

## 📌 Overview

This hands-on demonstrates unit testing in Angular using Angular's testing framework. It includes testing standalone components, services, component inputs and outputs, lifecycle hooks, routing dependencies, and mocked services.

---

## ⚠️ Angular Version Note

> **This project is implemented using Angular 21.**
>
> The **Digital Nurture 5.0 Hands-On Exercise Book** is written for **Angular 20**. Since this project was originally created with **Angular 21**, some testing configurations and APIs differ slightly from the exercise book.
>
> The implementation was adapted to Angular 21 while maintaining the same functionality, objectives, and expected outcomes described in the hands-on.

---

## 🎯 Objectives

- Learn Angular unit testing.
- Configure TestBed for standalone components.
- Test Angular components and services.
- Mock dependencies such as Router and ActivatedRoute.
- Verify component behavior using Jasmine expectations.
- Test HTTP-related services using mocked data.
- Validate Angular lifecycle methods.
- Improve overall test coverage.
